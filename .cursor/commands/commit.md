# commit

현재 변경분을 스테이징하고 커밋한다. 기본 규칙은 홈의 `~/.cursor/commands/commit.md`와 동일하며, **이 프로젝트(my-ridings)**에서는 아래 모노레포 규칙을 추가로 적용한다.

## 커밋 메시지 규칙
- 가급적 **한글**로 작성한다. 한글이 부적절한 경우에만 영문 사용.
- **함축적**으로 변경사항을 한 줄로 표현한다.
- feature 라도 `feat:` 같은 prefix 는 붙이지 않는다.
- 리팩토링인 경우에만 `R:` prefix 를 붙여도 된다.
- 한 줄로 부족하면, 짧은 제목 다음 줄부터 `- ` 로 시작하는 부연 설명을 여러 줄 추가해도 된다.

## 이 프로젝트만: 모노레포 prefix
- **형식**: `[앱이름] 제목` (부연이 있으면 그대로 이어서)
- **앱이름**: `apps/` 하위 디렉터리명과 동일 (kebab-case). `[plan-my-route]`, `[strava-boost]` 등.
- 루트·공통 설정만 수정한 경우 prefix 없이 작성하거나 `[root]` 등으로 표기.
- 커밋 전에 변경 파일이 주로 어떤 앱인지 확인한 뒤 해당 prefix를 붙인다.

## 절차
1. `git status` / `git diff` 로 변경 내용 파악
2. 위 규칙에 맞춰 커밋 메시지 작성 (이 repo에서는 `[앱이름]` prefix 포함)
3. `git add` 후 `git commit -m "제목"` 실행. 부연이 있으면 `-m "제목" -m "- 내용1" -m "- 내용2"` 형태
4. 커밋 메시지와 수행한 명령을 사용자에게 알린다

## 예시
- `[plan-my-route] 주변 탐색 검색 범위를 경로+2km로 축소`
- `[plan-my-route] 스테이지 카드 환산 거리 표시 개선`
- `R: [plan-my-route] getShippers 호출부를 api 모듈로 이동`
- `[plan-my-route] 로그인 실패 시 에러 메시지 노출` + 본문: `- 토스트로 표시` / `- 3초 후 자동 닫힘`
- `turbo 버전 2.8.17로 고정` (루트만 변경 시)
